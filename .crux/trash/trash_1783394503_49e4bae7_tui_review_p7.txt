S4：_stream_response 里轮询 _queued_text 不够干净
问题

for kind,payload in send_stream → 每轮检查 _queued_text 是临时解法。

问题：

如果流很久不 yield，插话无法及时处理
queued_text 没有强类型
不能区分 correction / priority / cancel
修改文件
ui/tui_v2.py
core/control_plane.py
改法

不要检查 _queued_text，改检查 control_queue：

Python
运行
def _drain_control_events(self):
    while event := self.control_queue.pop_nowait():
        self._handle_control_event(event)

在每个 stream chunk 后：

Python
运行
self._drain_control_events()

如果底层工具支持 cancel token，也传进去。
