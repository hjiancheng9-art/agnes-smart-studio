1. 文件分类：哪些必须保留，哪些可清理

建议把项目文件分成 5 个等级。

T0：绝对保留，默认永不自动删除

这些是 CRUX 的“源头资产”。

类型	示例	原因
源代码	core/, ui/, agents/, skills/, tools/	项目主体
配置与契约	config/*.yaml, *.toml, pyproject.toml, .env.example	运行和部署依据
技能 manifest	skill_manifest.json, *.skill.json	技能治理入口
ADR / 方法论	docs/adr/, methodologies/, CWIM, ASM	架构决策不可丢
测试代码	tests/	自愈和回归基础
数据库 schema / migration	migrations/, schema.sql	状态结构依据
手写模板	templates/, workflow_templates/	人工沉淀资产
README / 文档	README.md, docs/	项目知识
锁文件	requirements.lock, poetry.lock, uv.lock	可复现依赖

策略：

T0 默认永不删除。
除非用户显式指定，并且进入人工确认。
T1：重要状态，默认保留但可归档

这些不是源码，但对追踪、恢复、审计很重要。

类型	示例	建议
EventLog	.crux/event_log.sqlite	保留主库，定期压缩/归档
Goal 记录	.crux/goals/*.json	completed 30-90 天后归档
Run trace	.crux/runs/	近期保留，旧的压缩
patch backup	.crux/backups/	7-30 天保留
quarantine 记录	.crux/quarantine.sqlite/json	长期保留摘要
Retro / error memory	.crux/retro/, .crux/error_memory/	保留摘要，清理原始大日志

策略：

不直接删除，先归档。
归档后再按 TTL 清理。
T2：可重新生成，适合定期清理
类型	示例	TTL 建议
Python 缓存	__pycache__/, .pytest_cache/, .mypy_cache/, .ruff_cache/	随时可删
构建产物	dist/, build/, *.egg-info/	随时可删
TUI 渲染缓存	.crux/cache/tui/	7 天
搜索索引缓存	.crux/cache/search/	7-30 天
embedding cache	.crux/cache/embeddings/	30-90 天，按版本失效
浏览器截图缓存	.crux/cache/browser/screenshots/	3-14 天
临时下载	.crux/tmp/downloads/	1-7 天
preview 图片	.crux/previews/	7-30 天
ComfyUI 临时 workflow	.crux/comfyui/tmp_workflows/	7-14 天

策略：

可自动清理。
先 dry-run，再 move-to-trash/quarantine，最后删除。
T3：生成资产，大但可能有价值

这类最危险：不能一刀切删除。

类型	示例	处理方式
ComfyUI 输出图	outputs/images/, ComfyUI/output/	按标记清理
生成视频	outputs/videos/	默认不删，只提示大文件
LoRA checkpoints	lora_runs/*/checkpoints/	保留 best/final，清理中间
LoRA samples	lora_runs/*/samples/	可按 TTL 清理
训练日志	lora_runs/*/logs/	压缩保留
数据集副本	datasets/processed/	可再生则清理，原始数据永不删
浏览器录屏	.crux/browser/recordings/	7-30 天
代码生成草稿	.crux/generated/	30 天

策略：

必须区分 raw / curated / generated / intermediate。
原始素材和人工精选永不自动删除。
中间产物可清理。
T4：临时垃圾，优先清理
类型	示例
.tmp, .bak, .old, .orig	
*.pyc, *.pyo	
~$*, .DS_Store, Thumbs.db	
崩溃 dump	
空目录	
未引用临时文件	
超时未完成的 partial download	
失败 run 产生的孤儿目录	

策略：

可自动清理，但仍建议先移动到 .crux/trash/。
2. 自动清理机制设计

我建议做一个模块：

core/cleanup_manager.py

不要写成简单脚本，而要做成 CRUX 的治理模块。

核心流程
scan
→ classify
→ protect check
→ dependency/reference check
→ risk score
→ dry-run report
→ move to .crux/trash
→ delayed delete
→ EventLog record

不要直接 delete。

推荐目录结构
.crux/
  cache/
  tmp/
  trash/
  cleanup/
    policy.yaml
    reports/
    allowlist.txt
    denylist.txt
  backups/
  event_log.sqlite

      - "logs/**/*.log"

  directory: ".crux/trash"

  max_delete_per_run: 500

3. 防误删安全策略

这是最关键的。建议至少做 12 道安全闸。

安全闸 1：永远禁止项目根外删除

所有路径必须 resolve 后确认在项目根内。

Python
运行
root = Path(project_root).resolve()
target = path.resolve()

if not str(target).startswith(str(root)):
    raise SecurityError("Refuse to clean path outside project root")
安全闸 2：拒绝跟随 symlink 删除外部文件

如果是 symlink：

默认只删除 symlink 本身，不删除指向目标。
如果 symlink 指向项目外，标记 high-risk。
安全闸 3：protected path 永不自动删除

即使匹配了 cleanup rule，只要命中 protected，就跳过。

例如：

core/**
ui/**
skills/**
docs/**
tests/**
methodologies/**
*.py
*.md
*.json

注意：.json 不能全局保护，否则 ComfyUI 临时 JSON 清不了。更好的策略是：

保护 docs/config/templates 下的 JSON
不保护 .crux/tmp 下的 JSON
安全闸 4：删除改为 move-to-trash

不要直接删。

原路径:
  outputs/tmp/a.png

移动到:
  .crux/trash/cleanup-20260707-0643/outputs/tmp/a.png

保留 manifest：

JSON
{
  "cleanup_id": "cleanup-20260707-0643",
  "original_path": "outputs/tmp/a.png",

例如：

单文件 > 500MB

必须提示：

4. 推荐实现：cleanup_manager.py 骨架

可以这样设计。

Python
运行
from dataclasses import dataclass
from pathlib import Path
import hashlib
import shutil
import time

@dataclass
class CleanupCandidate:
    path: Path
    rule: str
    risk: str
    size: int
    reason: str
    action: str
    protected: bool = False
    referenced: bool = False

@dataclass
class CleanupPlan:
    cleanup_id: str
    candidates: list[CleanupCandidate]
    total_bytes: int
    high_risk_count: int
    dry_run: bool = True

class CleanupManager:
    def __init__(self, project_root: Path, policy: dict, event_log=None):
        self.root = project_root.resolve()
        self.policy = policy
        self.event_log = event_log

    def scan(self) -> CleanupPlan:
        candidates = []
        for rule_name, rule in self.policy["cleanup_rules"].items():
            for pattern in rule["patterns"]:
                for path in self.root.glob(pattern):
                    candidate = self._classify(path, rule_name, rule)
                    if candidate:
                        candidates.append(candidate)

        candidates = self._apply_safety_checks(candidates)
        return self._build_plan(candidates)

    def apply(self, plan: CleanupPlan, confirmed: bool = False):
        if plan.high_risk_count > 0 and not confirmed:
            raise RuntimeError("High-risk cleanup requires confirmation")

        trash_dir = self.root / ".crux" / "trash" / plan.cleanup_id
        trash_dir.mkdir(parents=True, exist_ok=True)

        for c in plan.candidates:
            if c.protected or c.referenced:
                continue
            self._move_to_trash(c.path, trash_dir)

    def _classify(self, path: Path, rule_name: str, rule: dict):
        if not path.exists():
            return None
        if not self._inside_root(path):
            return None

        min_age_days = rule.get("min_age_days", 0)
        if not self._older_than(path, min_age_days):
            return None

        return CleanupCandidate(
            path=path,
            rule=rule_name,
            risk=rule.get("risk", "medium"),
            size=self._size(path),
            reason=f"matched rule {rule_name}",
            action=rule.get("action", "trash")
        )

    def _inside_root(self, path: Path) -> bool:
        try:
            path.resolve().relative_to(self.root)
            return True
        except ValueError:
            return False

    def _older_than(self, path: Path, days: int) -> bool:
        if days <= 0:
            return True
        age_seconds = time.time() - path.stat().st_mtime
        return age_seconds >= days * 86400

    def _size(self, path: Path) -> int:
        if path.is_file() or path.is_symlink():
            return path.stat().st_size
        return sum(p.stat().st_size for p in path.rglob("*") if p.is_file())

    def _move_to_trash(self, path: Path, trash_dir: Path):
        rel = path.relative_to(self.root)
        dest = trash_dir / rel
        dest.parent.mkdir(parents=True, exist_ok=True)
        shutil.move(str(path), str(dest))
5. 针对 CRUX 的特殊清理规则

6. 自动清理调度策略

不要每天强删。建议：

每次启动：只做轻量 scan，提示可清理空间
每周：自动 dry-run
每月：提示执行 cleanup apply
空间不足时：触发 emergency cleanup

Emergency cleanup 只清：

__pycache__
pytest/mypy/ruff cache
.crux/tmp
browser screenshots older than 3 days
failed temp workflows

不碰：

模型
LoRA
outputs
datasets
用户文档
源码

最终建议

CRUX 的自动清理不应该是简单的 delete tmp files，而应该是一个 Cleanup Governance System：

文件分类
→ 规则匹配
→ 安全检查
→ dry-run
→ move-to-trash
→ 可恢复
→ EventLog 审计

一句话：

能重建的才清，能恢复的才删；未知的一律不删，只提示。
