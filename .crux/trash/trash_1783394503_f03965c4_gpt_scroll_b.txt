已思考几秒

这个痛点不要从“让鼠标拖选时还能滚动”入手。终端环境里，鼠标选区、滚轮、TUI 鼠标事件、终端原生复制机制天然会冲突。正确设计应该是：

不要让用户复制“屏幕文本”，而是让用户复制“消息对象”。

也就是说，CRUX TUI 应该提供消息级复制、详情页复制、片段选择模式，而不是依赖终端原生鼠标拖选。

推荐方案：三层复制体系
1. 消息级复制：解决 80% 场景

每条消息都变成可操作对象。

聚焦某条消息时显示：

Assistant #42 · 216 lines

↑ / ↓      切换消息

m          复制当前消息 Markdown
s          进入片段选择模式
o          打开消息详情页

命令：

/copy last
/copy 42
/copy 42 markdown
/copy 42 plain

这样用户不需要鼠标拖完整个长回复，一键复制整条消息。

2. Message Detail：长消息详情页

对于超过一屏的消息，主聊天流不应该承担完整阅读和复制任务。

按 o 打开详情页：

┌─ Message Detail · Assistant #42 · 216 lines ─────────────┐

├───────────────────────────────────────────────────────────┤
│ 这里显示完整消息内容，可独立滚动                           │
│ ...                                                       │
└───────────────────────────────────────────────────────────┘

详情页特性：

↑ / ↓                 单行滚动
c                     复制全文
s                     选择片段

这个方案很稳，因为它不依赖终端原生鼠标选区。
