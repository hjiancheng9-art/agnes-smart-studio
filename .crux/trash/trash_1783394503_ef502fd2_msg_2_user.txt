## 最后两个问题

### Q1: 自愈系统的假修复检测——具体怎么设计？

你说 checksum pre/post state。具体来说，对于工具调用，pre/post state 是什么？
- 如果是文件操作：file hash before/after
- 如果是代码生成：syntax check + test pass
- 如果是网络请求：HTTP status + response validation

但大多数 CRUX 工具调用不是这么干净。比如 trm_route 调用 web_search——pre/post state 是什么？

我倾向于：不是每种工具都要假修复检测，而是只对高风险工具（patch_file、execute_plan、self_heal）做。你怎么看？

### Q2: 技能包 manifest —— 最小可行版本是哪几个字段？

你说技能包要像 PyPI 包一样有签名、版本锁、依赖图。但现在 CRUX 只有 51 个技能，过度设计会让贡献者望而却步。

够不够？要不要加签名（minisign/JWT）？如果加签名，验证在加载时做还是运行时做？
