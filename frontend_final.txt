2. 建议：重新写前端还是增量修改？

5. V2 已有 HUD，ComfyUI 智能体是否需要自己的前端？
结论：需要，但不应该和 V2 割裂
最推荐方案：
V2 HUD = 总控驾驶舱ComfyUI Agent Console = 专业工作台两者共享 API client / UI kit / job system
不要完全复用 V2 HUD，也不要完全孤立。

为什么不能只复用 V2 HUD？
V2 HUD 更像：
总控视角阶段流转任务状态全局调度多执行器监控
而 ComfyUI 智能体需要很多专业页面：
工作流模板库参数 schema 表单节点 / 模板详情LoRA 数据集审核LoRA caption 编辑训练 checkpoint 对比KSampler 参数解释图片 / 视频结果画廊
这些塞进 V2 HUD 会让总控界面臃肿。

为什么也不能完全独立？
因为 CRUX / V2 是总流程入口。
如果 ComfyUI 前端完全独立，会出现：
V2 看不到 ComfyUI job 状态ComfyUI 看不到 CRUX stage 上下文结果不能顺畅回传下一阶段用户在两个系统之间跳来跳去
所以正确关系是：
V2 HUD 调用 ComfyUI Agent Console 的能力ComfyUI Agent Console 可以作为 V2 的子页面 / 插件 / 独立路由
