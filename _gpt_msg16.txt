落地得很对。你现在已经把“方法论文档”变成了运行时约束，下一步就应该把它继续下沉到：

TaskSpec → WorkflowIR → GraphCompiler → Validator → Executor

也就是让方法论不只是提醒 LLM，而是变成代码层的“工作流编译协议”。

1. 原则 3 的 Compiler 层怎么设计？

核心目标：

LLM 不直接生成最终 ComfyUI JSON。LLM 只生成 TaskSpec / WorkflowIR，GraphCompiler 负责确定性生成 ComfyUI workflow。

推荐分 4 层：

User Request
↓
TaskSpec             # 用户任务规格
↓
WorkflowIR           # 抽象工作流中间表示
↓
GraphCompiler        # 编译为 ComfyUI API JSON
↓
Validator            # 编译后验证
↓
Executor             # 提交 ComfyUI
1.1 TaskSpec：用户需求的结构化结果

1.4 GraphCompiler：确定性编译器

GraphCompiler 不应该“猜”。它应该根据：

WorkflowIR
NodeOntology
MotifLibrary
ComfyUI object_info schema
ModelInventory
ParameterEngine

确定性生成 ComfyUI API workflow。

建议结构：

compiler/
├── task_to_ir.py              # TaskSpec → WorkflowIR
├── graph_compiler.py          # WorkflowIR → ComfyUI JSON
├── node_factory.py            # role/motif → concrete node
├── connection_builder.py      # IR connection → ComfyUI input link
├── param_resolver.py          # 默认值 / 用户值 / 推荐值合并
├── model_resolver.py          # 选择 ckpt / lora / vae / controlnet
├── motif_registry.py          # motif 定义库
├── compile_context.py         # 编译上下文
└── compile_errors.py

核心接口：

Python
运行
class GraphCompiler:

        self,
        ir: WorkflowIR,
        context: CompileContext,

        ...

1.5 编译流程
1. 读取 WorkflowIR
2. 展开 motif
3. 为每个 component 分配 ComfyUI node_id
4. 根据 role 选择具体 class_type
5. 合并参数：用户参数 > IR 参数 > motif 默认 > parameter_engine 默认
6. 根据 object_info 校验输入字段
7. 建立连接
8. 自动补必要节点
9. 自动补输出节点
10. 生成 ComfyUI API JSON
11. 交给 Validator

伪代码：

Python
运行
def compile(ir: WorkflowIR, ctx: CompileContext) -> CompiledWorkflow:
    graph = {}
    node_map = {}

    for component in ir.components:
        node_def = ctx.motif_registry.resolve(component.motif, component.role)
        node_id = ctx.id_allocator.next()
        node_map[component.id] = node_id

        params = ctx.param_resolver.resolve(
            component=component,
            node_def=node_def,
            task_params=ir.params,
            user_params=ctx.user_params,
        )

        graph[node_id] = {
            "class_type": node_def.class_type,
            "inputs": params,
        }

    auto_connections = ctx.connection_builder.infer(ir, node_map, graph)
    for conn in auto_connections:
        target_id = node_map[conn.to_component]
        source_id = node_map[conn.from_component]
        graph[target_id]["inputs"][conn.to_port] = [source_id, conn.from_port_index]

    graph = ctx.output_builder.ensure_outputs(ir, node_map, graph)

    return CompiledWorkflow(
        workflow=graph,
        ir=ir,
        node_map=node_map,
        warnings=[],
        metadata={"compiler_version": "1.0.0"},
    )

不要把 motif 写死在 Python 里，建议 YAML 化。

    class_type: CheckpointLoaderSimple

      clip: CLIP
      vae: VAE

  - id: positive

      text: STRING

      conditioning: CONDITIONING

  - id: negative
    role: negative_prompt_encoder
    class_type: CLIPTextEncode
    inputs:
      clip: CLIP
      text: STRING
    outputs:
      conditioning: CONDITIONING

  - id: latent
    role: latent_init
    class_type: EmptyLatentImage
    outputs:
      latent: LATENT

  - id: sampler
    role: sampler
    class_type: KSampler
    inputs:
      model: MODEL
      positive: CONDITIONING
      negative: CONDITIONING
      latent_image: LATENT
    outputs:
      latent: LATENT

  - id: decode
    role: vae_decode
    class_type: VAEDecode
    inputs:
      samples: LATENT
      vae: VAE
    outputs:
      image: IMAGE

  - id: save
    role: image_output
    class_type: SaveImage
    inputs:
      images: IMAGE

default_connections:
  - from: checkpoint.model
    to: sampler.model
  - from: checkpoint.clip
    to: positive.clip
  - from: checkpoint.clip
    to: negative.clip
  - from: checkpoint.vae
    to: decode.vae
  - from: positive.conditioning
    to: sampler.positive
  - from: negative.conditioning
    to: sampler.negative

    to: sampler.latent_image

    to: save.images

这样智能体“学习”到的 2151 个连接模式，就可以沉淀成 motif 库，而不是继续作为统计结果。

2. Validator 应该检查哪些断言？

Validator 要分 5 层：

L1 JSON 结构校验
L2 ComfyUI 节点 schema 校验
L3 图拓扑校验
L4 运行时资源校验
L5 业务语义校验
L1：JSON 结构断言

检查 ComfyUI workflow 是否是合法 API 格式。

必须是 dict
每个 key 是 node_id
每个节点必须有 class_type
每个节点必须有 inputs
inputs 必须是 dict
连接格式必须是 [node_id, output_index]
node_id 不重复
不能有空节点
不能有不可序列化对象
