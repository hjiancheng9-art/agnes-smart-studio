## 一、Agnes 的核心定位

Agnes 不是单纯的聊天模型，它更像：

```text
文本模型 + 图像模型 + 视频模型 + 模板化创作工具 + Agent 工作流
```

视觉生产维度：

```text
Agnes Image = 高信息密度生图 / 改图 / 商业视觉
Agnes Video = 文生视频 / 图生视频 / 多图视频 / 关键帧过渡
Agnes App = 模板化图像视频创作入口
Agnes API = OpenAI 风格接入的自动化创作管线
```

一句话总纲：

```text
Agnes 生图 = 高信息密度视觉生成 + 构图保留式编辑。
Agnes 生视频 = 用短镜头、参考图、多图和关键帧控制运动。
```

Agnes 的稳定创作方式不是"一句话生成大片"，而是：

```text
先关键帧，后短镜头，再剪辑。
```

## 三、Agnes 生图方法论

### 3.1 Agnes Image 2.1 Flash 适合什么

Agnes Image 2.1 Flash 更适合高信息密度图像、复杂构图、细节密集场景、多层级视觉资产、商业视觉、广告主图、图生视频关键帧、构图保留式编辑。它的强项不是"极简一句话抽卡"，而是画面元素多、视觉层次多、构图复杂、细节密集、需要清楚主次关系、需要稳定进入视频环节。

最适合场景：复杂场景概念图、高信息密度海报、产品广告图、活动主视觉、电商产品图、图生视频关键帧、场景重绘、风格转换、背景替换、复杂建筑/城市/奇幻场景、多层级视觉资产。

### 3.2 生图总公式

```text
用途 + 图像类型 + 主体 + 场景/环境 + 视觉层级 + 构图 + 光线 + 色彩 + 材质 + 风格 + 质量要求 + 保留项 + 限制项 + 输出尺寸
```

### 3.3 生图的 10 个核心维度

**1. 用途** — 先写用途，模型才知道画面该服务什么。常见：电商主图、品牌海报、社交媒体封面、短视频首帧、电影关键帧、概念设定图、产品广告图、角色资产图、场景资产图。

**2. 主体** — 必须可识别、可锁定。写清：谁、几个、在哪、外观、颜色、材质、姿态、动作状态、标志性特征。

**3. 场景** — Agnes Image 2.1 适合高信息密度场景，但必须给视觉层级。拆成前景、中景、背景、主体活动区域、空间深度、环境氛围。

**4. 视觉层级** — 复杂画面必须写主次。说明主视觉是什么、第二层辅助视觉是什么、背景环境元素是什么、哪些元素不能抢主体、观众第一眼应该看哪里。

**5. 构图** — 常用：中心构图、三分法、低角度仰拍、俯视构图、宽幅远景、竖版海报构图、电商居中构图、前景遮挡、纵深透视、对称构图、引导线构图。

**6. 光线** — 写方向、强弱和质感。常用：柔和影棚光、金色时刻、低照度电影光、霓虹反光、侧逆光、强轮廓光、自然窗光、体积光、高光滑过、冷暖对比光。

**7. 色彩** — 写主色、辅色、对比和情绪。常用：黑金高级感、白灰极简感、深蓝科技感、青橙电影调色、低饱和冷色、暖米色生活方式、银白未来感、紫蓝霓虹感。

**8. 材质** — 产品图和高端视觉必须逐件写材质。常用：透明玻璃、厚玻璃折射、拉丝金属、磨砂金属、哑光纸、丝绸、陶瓷、湿润柏油路、水滴、烟雾、皮革、碳纤维、半透明树脂。

**9. 风格** — 一次只保留一个主风格。禁止风格堆叠：赛博朋克+水墨+吉卜力+皮克斯+写实摄影+油画。

**10. 限制项** — 决定画面不乱。常用：不要水印、不要多余文字、不要额外人物、不要产品变形、不要主体形状改变、不要杂乱背景、不要过曝、不要廉价塑料感、不要低清晰度、不要错误 logo、不要多手指、不要畸形五官。

## 四、Agnes 图生图 / 改图方法论

Agnes Image 2.1 Flash 支持 image-to-image，重点是**保留原始构图和主体布局**。

### 4.1 图生图总公式

```text
保持什么 + 只改什么 + 改成什么 + 新元素如何融合 + 禁止改变什么
```

### 4.2 五种改图模式

**换背景：** 保持人物脸部/发型/服装/姿势/构图不变，只替换背景，新背景匹配原图透视/光线/景深/色温。

**产品场景合成：** 保持产品形状/比例/logo位置/颜色/材质不变，只把产品放入新场景，产品阴影/反射/透视必须匹配新环境。

**风格转换：** 保持主体/构图/相机角度/场景布局不变，只改变整体视觉风格。

**删除物体：** 删除指定物体，自然补全被遮挡背景，保持光线/透视/材质/阴影/构图不变。

**添加物体：** 在指定位置添加物体，匹配原图透视/光线/阴影/接触关系，保持其他内容不变。

## 五、Agnes 生视频方法论

### 5.1 Agnes Video V2.0 支持的模式

Text-to-video / Image-to-video / Multi-image video / Keyframe animation / Motion control / Visual consistency / Cinematic output。

最佳用法：用 Agnes Image 做关键帧 → 用 Agnes Video 让关键帧动起来 → 用多图控制参考 → 用关键帧控制过渡 → 用短镜头剪成完整视频。

### 5.2 视频总公式

```text
Subject + Action + Scene + Camera Movement + Lighting + Style
```

扩展为完整生产模板：

```text
生成一个 [时长] 视频，[比例]，[分辨率]。
主体：[主体描述]
动作：[一个明确动作，从开始到结束]
场景：[地点、时间、天气、前景、中景、背景]
摄影机：[景别、视角、一个主运镜]
光线：[光源方向、强弱、色温、变化]
风格：[写实/电影/广告/动画/科幻/纪录片]
结尾：[最后一帧停在哪里]
限制：不要快速切镜，不要主体变形，不要添加多余人物，不要水印。
```

## 六、Agnes 文生视频 T2V 方法论

文生视频适合空镜、环境氛围、不要求固定角色的短镜头、广告概念片、电影感镜头、产品展示概念、社媒短视频。最重要的是说清谁、在哪里、做什么、镜头怎么拍、最后停在哪里。T2V 不适合承担强身份一致性任务，如果角色/产品/服装/具体场景必须一致，优先使用 I2V、多图视频或关键帧动画。

## 七、Agnes 图生视频 I2V 方法论

### 7.1 核心逻辑

参考图负责：主体、构图、风格、第一帧、身份、产品形状、场景基调。提示词负责：动作、运镜、光线变化、背景轻微变化、结尾状态。不要重复描述参考图所有静态内容，重点写"怎么动"。

### 7.2 图生视频模板

```text
基于参考图生成 [时长] 视频。
保持参考图中的主体身份、构图、服装、场景、光线、色彩和风格不变。
只添加以下运动：主体 [动作]、摄影机 [运镜]、背景 [轻微变化]、光线 [轻微变化]。
结尾停在：[尾帧状态]。
不要改变主体外貌，不要改变服装，不要改变产品形状，不要添加多余人物，不要快速切镜。
```

## 八、Agnes 多图视频方法论

多图视频必须一图一职责：图1=主体身份参考、图2=服装/产品/道具参考、图3=场景参考、图4=风格和色彩参考。自然融合参考图不要机械拼贴，保持主体身份/服装主色/产品形状和整体风格一致。

---

## 九、Agnes 关键帧动画方法论

关键帧逻辑：首帧=起点、尾帧=终点、中间帧=过渡控制、提示词=运动方式和视觉连续性。适合产品开盒、人物转身、远景推进到近景、日夜转换、装置启动、A状态变成B状态、空间穿越、物体展开。

## 十、Agnes 参数方法论

**图像参数：** 常用尺寸 1024×768（横版广告）、768×1024（竖版海报）、1024×1024（方图）、1536×864（宽屏关键帧）、864×1536（竖屏首帧）。

**视频参数：** 分辨率层级 480p/720p/1080p，比例 16:9/9:16/1:1/4:3/3:4。

**帧数与时长：** `seconds = num_frames / frame_rate`。num_frames ≤ 441 且必须符合 8n+1。常用：49帧/24fps≈2秒、121帧/24fps≈5秒、241帧/24fps≈10秒、441帧/24fps≈18秒。

**Seed 与 Negative Prompt：** 同一系列角色/产品广告固定 seed，探索变化时更换 seed，重跑必须记录 seed。常用 Negative：low quality、blurry、distorted face、extra fingers、product deformation、text、watermark、fast cuts、shaky camera、extra people、wrong logo、deformed object、inconsistent lighting。

## 十一、Agnes 项目级工作流

**最稳通用流程：** 写创意 → 拆分镜 → 每镜头一个主动作 → Agnes Image 生成关键帧 → 图生图修正 → Agnes Video 图生视频 → 复杂过渡用关键帧动画 → 多主体一致性用多图参考 → 多 seed 候选 → 选最稳版本 → 剪辑成片。

**产品广告流程：** 产品图 → Agnes Image 高端广告关键帧 → 图生图锁产品形状 → Agnes Video 图生视频 → 产品不动只让摄影机动 → 高光滑过 → 结尾特写 → 剪辑成 10-15 秒广告。

**角色短剧流程：** 角色设定 → Agnes Image 角色图 → 不同角度/表情 → 场景关键帧 → Agnes Video 图生视频 → 每镜头短动作 → 剪辑 → 配音 → 字幕。

**社媒短视频 15 秒结构：** 0-2s 视觉钩子 → 2-5s 主体出现 → 5-9s 动作推进 → 9-12s 视觉高潮 → 12-15s 结尾记忆点。

## 十三、Agnes 常见翻车与修正

| 问题 | 原因 | 修正 |
|------|------|------|
| 画面杂乱 | 主次没写清楚 | 写主视觉/辅助元素/背景三层，辅助不能抢主体 |
| 图生图改太多 | 只写改什么，没写保留什么 | 保持主体/构图/视角/光线/布局不变，只改一个点 |
| 视频动作混乱 | 一个镜头动作太多 | 一镜一动作，静止→动作→停住 |
| 镜头乱飞 | 运镜词太多 | 单一连续镜头，只做缓慢推进 |
| 产品变形 | 无产品锁 | 保持形状/比例/logo/颜色/材质不变，只让摄影机动 |
| 人物变脸 | 无参考图/动作太大 | 用图生视频，保持脸部/发型/服装/比例不变 |
| 关键帧跳切 | 过渡描述不连续 | 平滑连续自然过渡，相同主体/场景/光线逻辑 |

## 十四、Agnes 质量评估与评分门禁表

生成完成不是可用。观感不是证据。喜欢不是证据。只有评分表 + 失败项记录 + 下一步决策，才是生产证据。

### 14.1 生图质量评分表

| 维度 | 评分 1–10 | 门禁线 | 低于门禁的处理 |
|------|----------:|------:|---------------|
| 主体准确度 | | 8 | 主体错，不得进入下一环节，必须重跑或改图 |
| 视觉层级 | | 8 | 主次不清，不得做复杂图/视频首帧 |
| 构图完成度 | | 8 | 构图乱，不得进入 I2V |
| 光线质量 | | 7 | 光线可修则修，不可修重跑 |
| 材质真实度 | | 7 | 产品图建议 ≥8，低于门禁需修图 |
| 风格统一性 | | 8 | 风格漂移需删冲突风格词重跑 |
| 细节密度 | | 7 | 复杂图细节不足需补层级，不得堆元素 |
| 背景干净度 | | 7 | 背景抢主体需降复杂度或重跑 |
| 商业可用性 | | 8 | 商业图低于 8 不得作为主图 |
| 是否适合图生视频 | | 8 | 低于 8 不得进入 I2V |

### 14.2 生图评分决策

```text
平均分 < 7：淘汰或大幅重跑。
7 ≤ 平均分 < 8：只能作为参考图或备选，不能进入主流程。
8 ≤ 平均分 < 9：可进入下一环节，但必须记录缺陷。
平均分 ≥ 9：可作为主版本或关键帧。
```

硬性规则：主体准确度 < 8 直接重跑；视觉层级 < 8 不得进入复杂图或视频首帧；构图完成度 < 8 不得进入 I2V；是否适合图生视频 < 8 不得进入 I2V。

### 14.3 生视频质量评分表

| 维度 | 评分 1–10 | 门禁线 | 低于门禁的处理 |
|------|----------:|------:|---------------|
| 主体一致性 | | 8 | 变脸/变形直接重跑或回到参考图 |
| 动作清晰度 | | 8 | 动作混乱先拆镜头，不得继续重跑同 prompt |
| 动作自然度 | | 7 | 动作僵硬可缩短时长或降低动作幅度 |
| 运镜稳定度 | | 8 | 镜头乱飞不得进入剪辑 |
| 构图连续性 | | 8 | 构图漂移需回到首帧或镜头合同 |
| 光线连续性 | | 7 | 光线跳变需弱化光线变化或重做关键帧 |
| 产品/角色保持度 | | 8 | 产品变形/人物变脸不得进入主剪辑 |
| 结尾可剪辑性 | | 8 | 无稳定尾帧不得进入主轨 |
| 是否适合接下一个镜头 | | 8 | 不适合接镜头需重写尾帧状态 |
| 是否值得重跑 | | 7 | 低于 7 淘汰，不继续消耗重跑次数 |

### 14.4 生视频评分决策

```text
平均分 < 7：淘汰，不再继续当前方向。
7 ≤ 平均分 < 8：可作为参考或局部素材，不进入主剪辑轨。
8 ≤ 平均分 < 9：可进入剪辑候选，但记录缺陷。
平均分 ≥ 9：可作为主镜头素材。
```

硬性规则：主体一致性 < 8 不得进入剪辑；运镜稳定度 < 8 不得进入剪辑；产品/角色保持度 < 8 不得进入剪辑；结尾可剪辑性 < 8 不得进入主轨。

### 14.5 关键帧动画质量评分表

| 维度 | 评分 1–10 | 门禁线 | 低于门禁的处理 |
|------|----------:|------:|---------------|
| 首尾主体一致性 | | 8 | 主体不一致，重做关键帧 |
| 首尾空间连续性 | | 8 | 空间跳变，添加中间帧或重做尾帧 |
| 过渡自然度 | | 8 | 过渡跳切，缩短变化或加中间帧 |
| 光线连续性 | | 7 | 统一色温和光源方向 |
| 材质连续性 | | 8 | 产品/服装材质变化需重做 |
| 动作逻辑清晰度 | | 8 | 起点终点不清，重写镜头合同 |
| 结尾稳定性 | | 8 | 结尾不稳，不进剪辑 |

### 14.6 剪辑进入评级

| 等级 | 条件 | 用法 |
|------|------|------|

| D 级 | 评分 < 7 | 淘汰 |

## 十五、Agnes 最强生产流程

```text
1. 写创意 → 2. 拆分镜 → 3. 每镜头一个动作 → 4. Agnes Image 关键帧 →
5. 图生图修关键帧 → 6. 首帧评分 → 7. Agnes Video 图生视频 → 8. 视频评分 →
9. 多图控制一致性 → 10. 关键帧动画做复杂转场 → 11. 多 seed 候选 →
12. 重跑/淘汰/入剪 → 13. 剪辑成片 → 14. 版本归档
```

## 十六、阶段性总结：Agnes 基础方法论

Agnes 的本质不是"一句话生成完整大片"，而是用 Agnes Image 做高信息密度视觉和关键帧、用 Agnes Video 做短镜头运动、用多图和关键帧控制一致性、最后靠剪辑成片。

基础口诀：生图看视觉层级、改图看保留项、复杂图看主次关系、图生视频看参考图、多图视频看分工、关键帧动画看过渡逻辑、视频质量看一镜一动作、商业成片看剪辑。

## 十七、强制铁律

### 铁律 1：没有用途，不生图
```
NO IMAGE WITHOUT PURPOSE
```
每张图必须先明确用途。禁止随便生成一张好看的图、先出图再想用途。用途不清，画面就没有取舍标准。

### 铁律 2：没有视觉层级，不做复杂图
```
NO COMPLEX IMAGE WITHOUT VISUAL HIERARCHY
```
复杂画面必须写清楚主视觉、第二视觉、背景信息。禁止大量元素堆叠、主体和背景同等重要。

### 铁律 3：没有保留项，不改图
```
NO IMAGE EDIT WITHOUT PRESERVATION RULES
```
改图前必须写清保持什么、只改什么、禁止改变什么。改图失败的根因通常不是"改得不够"，而是"没有规定不能改什么"。

### 铁律 4：没有合格首帧，不做图生视频
```
NO I2V WITHOUT QUALIFIED FIRST FRAME
```
首帧必须通过检查。首帧不合格不允许靠视频提示词补救，必须先回到 Image 修正。

### 铁律 5：没有一镜一动作，不生成视频
```
NO VIDEO WITHOUT ONE SHOT, ONE ACTION
```
每个镜头只能有一个主动作。动作复杂就拆成多个短镜头。

### 铁律 6：没有唯一运镜，不生成视频
```
NO VIDEO WITHOUT ONE CAMERA MOVEMENT
```
每个镜头只允许一个主运镜。运镜越少，视频越稳。

### 铁律 7：没有结尾状态，不生成视频
```
NO VIDEO WITHOUT ENDING FRAME
```
必须写最后一帧停在哪里、主体最后是什么状态、是否适合接下一个镜头。

### 铁律 8：没有质量评分，不进入剪辑
```
NO EDITING WITHOUT QUALITY SCORE
```
评分低于 7 不得进入剪辑。观感不是证据，评分表+失败项记录才是证据。

### 铁律 9：没有版本记录，不算生产流程
```
NO PRODUCTION WITHOUT VERSION TRACE
```
每次生成必须记录项目名、镜头编号、版本号、prompt、seed、参考图、参数、task_id、结果路径、评分、失败原因。不可复盘的生成不算生产流程。

## 十八、素材资产门禁

### 参考图门禁
参考图必须满足主体清晰、无遮挡、身份特征明确、光线方向稳定。不合格参考图不能通过 prompt 补救。

### 产品图门禁
产品必须锁定形状、比例、logo位置、颜色、主要材质。产品广告优先策略：产品尽量不动，让摄影机动、光线动、背景轻微动。

### 人物/角色图门禁
角色参考图必须锁定脸部、发型、服装、身体比例、气质、主色、标志性特征。

### 场景图门禁
场景必须锁定空间结构、光线方向、色彩氛围、前景/中景/背景关系、镜头可运动空间。

### 多图参考门禁
一张图一个职责。冲突时减少参考图而不是增加提示词。

### 关键帧门禁
首尾帧必须主体一致、服装一致、产品一致、材质一致、空间逻辑一致、光线连续。关键帧不连续不允许进入关键帧动画。

## 二十、重跑策略与版本纪律

### 重跑总原则
先判断失败类型，再决定回到哪一环。不要无脑加词、不要无脑换 seed、不要把参考图问题交给 prompt 解决、不要把首帧问题交给视频解决。

### 标准重跑流程
```text
第一轮：探索 seed，不改核心 prompt
第二轮：锁定构图，微调主体/光线/视觉层级
第三轮：锁定首帧，进入图生视频
第四轮：视频只改运动和运镜
第五轮：若身份/产品漂移，回到参考图或首帧
第六轮：若动作仍失败，拆镜头而不是继续加词
```

### 图生视频重跑策略
| 问题 | 策略 |
|------|------|
| 主体变形 | 降低动作幅度、缩短时长、让主体静止让摄影机动 |
| 人物变脸 | 回到身份参考图、减少动作、减少转头 |
| 产品变形 | 产品保持静止、只让高光和摄影机动、缩短镜头 |
| 镜头乱飞 | 只保留一个运镜、改成 slow push-in/locked camera |
| 动作混乱 | 拆成多个镜头、每镜头只保留一个动作 |

### 版本命名规范
```text
项目名_镜头号_模式_版本_评分
```

### 重跑红旗
| 想法 | 正确现实 |
|------|---------|
| 再加几个词应该会好 | 先判断失败属于图、视频、参考图还是关键帧 |
| 动作乱了就继续重跑 | 先拆成一镜一动作 |
| 人物变脸就补提示词 | 回到身份参考图 |
| 产品变形就继续 I2V | 回到产品首帧或让产品静止 |
| 这个版本有点问题但能剪 | 先评分，低于 7 不进剪辑 |

## 二十一、Agnes API 工程纪律

### API 铁律
```text
没有 task_id，不算生产记录。
没有 payload，不算可复盘。
没有 seed，不算可重跑。
没有失败日志，不算有效排错。
没有版本表，不算项目生产。
```

### API 请求前记录
每次创建任务前必须记录：项目名、镜头编号、生成模式、prompt、negative prompt、seed、参考图 URL、关键帧 URL、尺寸、帧数、帧率、目标时长、目标比例、调用时间。

### API 创建任务后记录
创建任务后必须保存：task_id、video_id、status、created_at、原始请求 payload、API 返回 payload。禁止只保存最终视频不保存任务 ID。

### 轮询纪律
轮询任务时必须记录：status、progress、error、video_url、duration/seconds、size、completed_at。如果 API 返回的时长/尺寸与本地计算不同，以 API 返回值为准。

### 失败任务记录
必须记录失败时间、错误码、错误信息、原始请求、参考图、参数、是否重试、重试改了什么。失败不可复盘就无法优化。

### 重跑必须记录变更点
每一次重跑必须写：这次改了什么、为什么改、预期解决什么问题、实际是否解决、评分变化。

### API 版本表
```text
项目 | 镜头 | 模式 | 版本 | task_id | video_id | seed | 参数 | 评分 | 状态 | 备注
```

## 二十二、失败诊断树

### 失败归因总表
```text
画面问题 → 回到 Image prompt/图生图
主体问题 → 回到参考图/主体锁定
构图问题 → 回到视觉层级/构图描述
运动问题 → 回到 Video prompt
身份漂移 → 回到参考图/多图分工
产品变形 → 回到产品首帧/降低运动
过渡跳切 → 回到关键帧连续性
剪辑不顺 → 回到镜头合同/结尾状态
```

### 人物变脸诊断树
```text
人物变脸
├─ 首帧已经不像 → 回到 Image/图生图，重做身份参考
├─ 首帧正确，视频中变脸
│  ├─ 动作幅度过大 → 降低动作
│  ├─ 转头过多 → 改成轻微抬头/固定角度
│  └─ 时长过长 → 缩短镜头
├─ 多图参考冲突 → 减少参考图
└─ 关键帧中变脸 → 重做尾帧
```

### 产品变形诊断树
```text
产品变形
├─ 首帧已变形 → 回到 Image/图生图修产品图
├─ 视频中变形
│  ├─ 产品自己在动 → 改为产品静止
│  ├─ 镜头太激烈 → 改为 slow push-in
│  └─ 时长太长 → 缩短
├─ logo 漂移 → 固定产品正面，减少旋转
└─ 材质变了 → 回到产品图，强化材质锁定
```

### 镜头乱飞诊断树
```text
镜头乱飞
├─ 运镜词太多 → 只保留一个运镜
├─ prompt 里有 dynamic/dramatic/handheld → 删除冲突词
├─ 动作和运镜同时复杂 → 固定主体只动镜头
└─ 时长过长 → 缩短为 2–5 秒
```

### 关键帧跳切诊断树
```text
关键帧跳切
├─ 首尾视角差太大 → 添加中间帧
├─ 首尾主体不一致 → 重做尾帧
├─ 光线差太大 → 统一色温和光源方向
└─ 过渡描述太少 → 明确"平滑连续过渡"
```

### 画面杂乱诊断树
```text
画面杂乱
├─ 主体不明确 → 重写主体和视觉焦点
├─ 辅助元素太多 → 删除 30–50% 元素
├─ 背景抢主体 → 降低背景对比和亮度
├─ 光太多 → 只保留一个主光源
└─ 风格词冲突 → 只保留一个主风格
```

## 二十四、完成前验证与质量门函数

### 完成门函数
在声明任何图像/视频可用前，必须完成：检查是否符合用途 → 检查主体是否准确 → 检查构图是否符合需求 → 检查风格是否统一 → 检查是否有变形/错字/多余主体 → 对照质量评分表打分 → 决定重跑/修图/进入视频/进入剪辑/淘汰。

### 生图完成标准
主体准确度 ≥ 8、视觉层级 ≥ 8、构图完成度 ≥ 8、光线质量 ≥ 7、材质真实度 ≥ 7、风格统一性 ≥ 8、适合图生视频 ≥ 8、无明显变形、无错误文字、无水印。低于标准不得进入图生视频。

### 图生视频完成标准
主体一致性 ≥ 8、动作清晰度 ≥ 8、运镜稳定度 ≥ 8、结尾可剪辑性 ≥ 8、无快速切镜、无严重变形、无多余主体。低于标准不得进入主剪辑轨。

### 剪辑进入标准
A 级（评分 9+）：主素材，可直接使用。B 级（评分 8+，有小问题但可遮挡）：备选素材。C 级（评分 7+）：只能参考。D 级（低于 7）：淘汰。

## 二十五、Agnes 视觉生产宪法

### 总流程宪法
```text
创意 → 分镜 → 镜头合同 → 资产门禁 → Agnes Image 关键帧 →
图生图修正 → 首帧评分 → Agnes Video 短镜头 → 视频评分 →
重跑/淘汰/入剪 → 剪辑 → 成片复查 → 版本归档
```

### 总铁律
```text
没有用途，不生图。
没有视觉层级，不做复杂图。
没有保留项，不改图。
没有合格首帧，不做图生视频。
没有一镜一动作，不生成视频。
没有唯一运镜，不生成视频。
没有结尾状态，不进入剪辑。
没有质量评分，不声明可用。
没有版本记录，不算生产流程。
```

### 最终判断标准
一张图是否成功看：是否服务用途、主体是否准确、主次是否清楚、构图是否稳定、是否适合进入下一环节。一个视频是否成功看：动作是否单一、运镜是否稳定、主体是否保持、结尾是否可剪辑、是否能接上下镜头。一个项目是否成功看：资产是否可复用、镜头是否可剪辑、版本是否可追踪、失败是否可诊断、成片是否稳定。

### 最终口诀
```text
生图先定用途。复杂图先定主次。改图先定保留项。视频先定首帧。
一镜只做一件事。多图只做一类参考。关键帧只做连续过渡。
重跑必须有原因。入剪必须有评分。成片必须可复盘。
```
