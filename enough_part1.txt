下面给你一个很实用的判断标准：

清债不是清到“没有债”，而是清到“债不会阻塞交付、不会放大风险、不会拖慢下一轮迭代”。

你们 v0.4.0、91 测试全绿后，已经不该继续无限清债。现在应该进入 “阈值达标就停” 模式。

总判断
够了 = 新人能接手 + 小白能看懂 + 本地执行不裸奔 + 参数有出处 + 核心路径有测试 + 出问题能定位

如果继续清债不能明显改善这 6 件事，就该停。

1. 体验债：够了的阈值

你们现在有：

UserFacingResult
中文错误翻译
Note 节点

这已经达到 v0.4.0 的够用线。

停手阈值

做到下面这些就够：

成功时：
- 用户知道生成了什么工作流
- 用户知道怎么拖进 ComfyUI 运行
- 用户知道输出在哪里
- 用户不用看 raw JSON 才能理解结果

失败时：
- 错误有中文标题
- 有中文原因
- 有明确下一步建议
- 不暴露 Python stack trace 给小白
- technical_report 仍保留给开发者
不要继续卷的东西
彩色 CLI 输出：可做，但不是 P0
复杂 UI：不要现在做
语音提示：完全没必要
动画进度条：除非你已经有真实执行进度
体验债停止标准
小白看到结果后，只需要知道：
1. 能不能跑
2. 怎么跑
3. 失败了怎么办

超过这个，就不是清债，是做产品包装。

2. 安全债：够了的阈值

你们现在有：

class_type 白名单
子进程白名单
路径校验

对于 本地 ComfyUI workflow compiler，这已经是合理安全基线。

停手阈值

做到下面这些就够：

workflow 送 /prompt 前：
- class_type 必须在 object_info / allowlist 中
- workflow 必须通过 validator
- workflow 必须通过 quality_gate
- workflow 必须通过 safety_gate
- 禁止未知节点
- 禁止可疑路径逃逸

子进程：
- shell=False
- 只允许白名单命令
- 必须 timeout
- 不拼接用户输入命令

路径：
- 所有读取限制在 ComfyUI 根目录或用户显式授权目录内
- 禁止 .. 路径逃逸
- 禁止跟随危险 symli
