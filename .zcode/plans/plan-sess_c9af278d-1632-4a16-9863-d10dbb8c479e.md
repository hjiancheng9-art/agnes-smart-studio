## 方案 A：完全解耦 — Worker 和 UI 之间加 Chunk Buffer

### 核心思路

Worker 线程不再调用任何 UI 函数（`stream_append`、`thinking_panel.append` 等）。改为写入线程安全的 chunk 队列，渲染周期在 UI 线程消费。

```
Worker 线程                          UI 线程 (ptk render)
─────────                          ─────────
chunk → [_chunk_queue]  ──→  _drain_chunks() on before_render
         under lock              → stream_append(text)
         + invalidate()          → thinking_panel.append(text)
                                 → done → _reduce(StreamDone())
```

### 改动清单

**1. `__init__` — 加 chunk buffer**
```python
self._chunk_queue: list[tuple] = []
self._chunk_lock = threading.Lock()
```

**2. `_fx_run_stream` — worker 只写队列**
- 初始调用（`stream_start`、`clear`）已在 UI 线程，直接调用，不用 `_threadsafe_call`
- Worker 内：`with _chunk_lock: _chunk_queue.append(("text", payload))` + `_app.invalidate()`
- Worker 结束：`_chunk_queue.append(("done",))` + `_app.invalidate()`
- 移除所有 `_threadsafe_call(stream_append)` 和 `_threadsafe_call(thinking_panel.append)`
- 异常路径：`_chunk_queue.append(("error", msg))`

**3. 新增 `_drain_chunks()` — UI 线程消费**
```python
def _drain_chunks(self):
    with self._chunk_lock:
        chunks, self._chunk_queue = self._chunk_queue, []
    for kind, *args in chunks:
        if kind == "text":   self.message_pane.stream_append(args[0])
        elif kind == "thinking": self.thinking_panel.append(args[0])
        elif kind == "error": self.message_pane.append_error(args[0])
        elif kind == "done": self._reduce(StreamDone())
```

**4. `_build_app` — 注册 drain 钩子**
```python
app.on_invalidate += lambda _: self._drain_chunks()
```

**5. `_restore_mouse` — 保留不动**
stream_append 现在只从 UI 线程调，写 stdout 安全。

### 收益
- Worker 永远不碰 Widget / stdout / Window 属性
- 所有 UI 操作都在 UI 线程
- `_restore_mouse` 不会和 ptk 渲染器抢 stdout（同线程串行）
- `_auto_scroll` 的 vertical_scroll 写入也在 UI 线程

### 风险
- `on_invalidate` 钩子需要确认 ptk 版本支持（≥3.0 有 events 系统）
- 首次渲染在 drain 之前可能需要初始化检查
