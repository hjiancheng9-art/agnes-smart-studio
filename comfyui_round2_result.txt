3. agent.py 拆分具体方案：如何避免循环导入
核心原则
你现在不能这样：
Python运行# agent.pyfrom agent_ops import diagnose_workflow# agent_ops.pyfrom agent import ComfyUIAgent
这必然循环。
正确依赖方向应该是：
contracts/types  ←  services/ops  ←  agent facade
也就是：
agent_types.py      不依赖 agent.pyagent_runtime.py    不依赖 agent.pyagent_services.py   不依赖 agent.pyagent.py            依赖上面所有模块
任何被 agent.py 导入的模块，都不允许再 import agent.py。

推荐目录结构
comfy_agent/├── agent.py                  # 薄 Facade├── agent_runtime.py           # 依赖容器├── contracts/│   ├── results.py             # SmartBuildResult / AutoFixResult 等│   ├── diagnostics.py         # WorkflowDiagnostic│   ├── requests.py            # BuildRequest / ModifyRequest│   └── errors.py              # 统一异常│├── services/│   ├── workflow_io.py         # load_workflow / diagnose_json 输入处理│   ├── diagnostic_service.py  # diagnose│   ├── edit_service.py        # modify_workflow│   ├── validation_service.py  # validate_workflow│   ├── template_service.py    # list/search/match/load templates│   ├── build_service.py       # smart_build│   ├── model_service.py       # required_models / inventory│   ├── report_service.py      # generate_report│   └── pipeline_service.py    # modify_and_validate / full pipeline│└── protocols/    └── runtime.py             # Protocol 类型，可选

4. brain + agent_flow LLM 去重方案
当前问题判断
现在大概率是这样：
brain.py:- 管 LLM provider- 发送请求- 处理 OpenAI-compatible API- 可能也拼 prompt / 解析结果agent_flow.py:- NL 输入- 判断意图- 生成 workflow- 也直接调用 LLM- 也处理 prompt / JSON / fallback
这会造成 LLM 逻辑重复。

正确分工
应该改成：
brain.py 只做 LLM 基础连接器agent_flow.py 只做业务编排中间新增 nl_planner / llm_service 负责结构化能力
推荐分层：
brain/├── client.py          # OpenAI-compatible transport，只负责请求├── schemas.py         # LLM 输出 schema├── prompts.py         # prompt 模板├── structured.py      # JSON schema 调用、解析、重试└── errors.py          # LLM 错误nl_flow/├── intent.py          # 意图识别├── planner.py         # NL → WorkflowPlan├── slot_filling.py    # 参数抽取├── repair.py          # 失败修复计划└── orchestrator.py    # 编排流程

brain.py 的职责
brain.py 或 brain/client.py 只负责：
加载模型配置调用 OpenAI-compatible API统一超时统一重试统一日志统一错误返回原始或结构化结果
它不应该知道：
ComfyUI workflow 结构节点怎么连模板怎么选模型文件怎么扫描V2桥接怎么返回CRUX怎么调度
示例：
Python运行class BrainClient:    def complete(self, messages, *, temperature=0.2, timeout=60) -> str:        ...    def complete_json(self, messages, schema, *, temperature=0.1, timeout=60) -> dict:        ...

agent_flow.py 的职责
agent_flow.py 不应该直接碰 provider。
它应该只做：
接收自然语言调用 NLPlanner拿到结构化计划调用 TemplateService / BuildService / ValidationService返回结果
示例：
Python运行class AgentFlow:    def __init__(self, planner, build_service, validation_service, report_service):        self.planner = planner        self.build_service = build_service        self.validation_service = validation_service        self.report_service = report_service    def run(self, user_message: str):        plan = self.planner.plan(user_message)        build_result = self.build_service.build_from_plan(plan)        validation = self.validation_service.validate(build_result.workflow)        return self.report_service.generate_pipeline_report(build_result, validation)

新增 NLPlanner：去重核心
把所有“自然语言 → 结构化计划”的 LLM 调用集中到一个地方：
Python运行class NLPlanner:    def __init__(self, brain_client):        self.brain = brain_client    def classify_intent(self, message: str) -> IntentResult:        ...    def extract_slots(self, message: str) -> SlotResult:        ...    def create_workflow_plan(self, message: str, context: dict) -> WorkflowPlan:        ...    def create_repair_plan(self, error_context: dict) -> RepairPlan:        ...
这样：
agent_flow.py 不再直接调用 LLMagent.py 不再直接调用 LLMsmart_build 不再直接调用 LLM所有 LLM 入口统一走 NLPlanner / BrainClient

推荐 LLM 调用链
最终应该是：
agent_flow.py↓NLPlanner↓BrainClient.complete_json()↓LLM Provider
业务执行链：
NLPlanner 输出 WorkflowPlan↓BuildService 根据 WorkflowPlan 构建 workflow↓ValidationService 验证↓EditService / AutoFixService 修复↓Executor 执行

LLM 输出必须结构化
不要让 LLM 返回自然语言再用正则硬拆。
建议所有关键 LLM 输出统一 schema。
例如：
Python运行@dataclassclass WorkflowPlan:    intent: str    template_name: str | None    positive_prompt: str    negative_prompt: str | None    aspect_ratio: str    required_models: list[str]    required_nodes: list[str]    steps: list[dict]    risk_level: str
LLM 返回：
JSON{  "intent": "text_to_image",  "template_name": "sdxl_basic_txt2img",  "positive_prompt": "warm photorealistic lunch scene...",  "negative_prompt": "low quality, blurry",  "aspect_ratio": "16:9",  "required_models": ["sdxl_base"],  "required_nodes": [],  "steps": [    {      "type": "load_template",      "name": "sdxl_basic_txt2img"    },    {      "type": "set_prompt",      "target": "positive",      "value": "warm photorealistic lunch scene..."    }  ],  "risk_level": "low"}

推荐最终调用关系
CRUX↓V2 Bridge↓ComfyUISubExecutorAdapter↓ComfyUIAgent Facade↓AgentFlow / BuildService / DiagnosticService↓NLPlanner↓BrainClient
注意：
BrainClient 在最底层，不能反过来 import agent_flow 或 agent.py。
正确依赖方向：
contracts↓brain↓nl_flow↓services↓agent facade↓v2 bridge
不要出现：
brain.py import agent_flow.pyagent_flow.py import agent.pyservices import agent.py
