2. 2024-2025 主流稳定节点 / 插件生态
2.1 最小必备层

建议你们把插件分成 4 档：

Tier 0: ComfyUI 原生节点
Tier 1: 稳定生产必备
Tier 2: 高级图像质量 / 控制 / 视频
Tier 3: 实验性 / 任务专用

Comfy Desktop 官方说明现在内置 ComfyUI Manager，可在应用内浏览、安装、更新 5000+ extensions；所以你们系统应优先适配 Manager / Registry，而不是手写 GitHub 安装逻辑。

2.3 IPAdapter / 参考图一致性

IPAdapter 仍是参考图风格迁移、主体迁移、FaceID 类工作流的核心方案之一。官方/社区描述里把它类比成“1-image LoRA”：可以把参考图里的主体或风格转移到生成结果中。
GitHub
+1

推荐你们内部建三个 recipe：

ipadapter_style_transfer
ipadapter_subject_reference
ipadapter_faceid_identity

不要把 IPAdapter 和 FaceSwap 混成一个能力。IPAdapter 是“参考条件注入”，ReActor 是“后处理换脸”，InstantID/PuLID 是“身份保持生成”。

2.4 ControlNet 新方案

comfyui-controlnet-aux 仍然是预处理器基础设施。它负责生成 canny、depth、openpose、lineart 等 hint image；它还提供 AIO Aux Preprocessor，可以把多个预处理器统一成下拉选项，但如果需要具体阈值，仍要使用独立节点。
GitHub
+1

高级 ControlNet 推荐加入 ComfyUI-Advanced-ControlNet，它支持更复杂的 ControlNet 应用、调度、mask、keyframe 类需求。官方/仓库说明它通过 Load Advanced ControlNet Model 和 Apply Advanced ControlNet 扩展标准 ControlNet 的能力。
GitHub

2.5 Flux 支持

Flux 已经不应该再当“实验模型”，而应当作为 2025 之后的高级图像工作流主线之一。ComfyUI 官方 Flux 教程覆盖了 Flux.1 dev / schnell、full version 和 FP8 checkpoint 两种工作流；官方也明确说 Flux 是 12B 参数模型，原始文件约 23GB，full version 效果最好但需要更多显存，FP8 checkpoint 资源需求较低但质量略降。
ComfyUI

对你们的 planner：

Python
运行
FLUX_BLUEPRINTS = [
    "flux_schnell_low_vram",
    "flux_dev_fp8_balanced",
    "flux_dev_full_high_quality",
    "flux_dev_gguf_low_vram",
    "flux_kontext_editing",
]

ComfyUI-GGUF 是低显存 Flux / SD3.5 Large 的关键补充；它支持 GGUF UNet loader，模型放在 ComfyUI/models/unet，预量化模型包括 flux1-dev、flux1-schnell、stable-diffusion-3.5-large 等。
GitHub
+1

2.6 视频节点生态

视频现在分两条路线：

A. 原生支持优先：
   Wan2.1 / Wan2.2 / LTX-2.3 原生模板

B. 自定义节点补充：
   VideoHelperSuite / AnimateDiff-Evolved / LTXVideo / WanVideoWrapper

ComfyUI 官方 Wan2.1 文档说明 Wan2.1 是阿里 2025 年 2 月开源的视频模型，包含 14B 和 1.3B 两个版本，覆盖 T2V/I2V，轻量版约 8GB VRAM 可运行。
ComfyUI

ComfyUI 官方 Wan2.2 文档说明 5B 版本在 ComfyUI 原生 offloading 下适合 8GB VRAM，并且 14B T2V/I2V 官方工作流需要 high-noise / low-noise 双 diffusion model 加 text encoder 和 VAE。
ComfyUI

LTX-2.3 已经是 ComfyUI 原生支持的现代视频路线，官方文档说明它支持 I2V/T2V、9:16 竖屏、改进的音频、更好的图生视频一致性，并给出 checkpoint、latent upscaler、text encoder 的模型目录结构。
ComfyUI
