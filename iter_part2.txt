v0.5.0 推荐范围

只做一个核心目标：

输入：104 个真实 workflow JSON
输出：5 个 mined production blueprint

不要一上来做完整 AI 自动生成。

最小实现：

1. workflow_parser.py
   解析每个 workflow 的 node types、links、输入输出节点

2. pattern_miner.py
   统计节点频率、节点共现、常见链路、阶段结构

3. blueprint_synthesizer.py
   生成 required_nodes / optional_nodes / stages

4. blueprint_review.py
   打分：覆盖率、复杂度、可执行性、硬件成本

5. production_blueprint_registry.py
   把 mined blueprint 注册进 compiler
v0.5.0 的验收标准
1. 能从 104 个 workflow 中自动挖出：
   - flux_pro blueprint
   - ltx_i2v_pro blueprint
   - ltx_audio_pro blueprint
   - wan_video_edit blueprint
   - flux_to_ltx_pipeline blueprint

2. 每个 blueprint 至少包含：
   - required_nodes
   - optional_nodes
   - common_edges
   - stage list
   - estimated_node_count
   - source_workflow_count
   - confidence_score

3. compiler 能优先选择 production blueprint

4. fallback 到原来的 7 节点基础蓝图

5. 生成的 workflow 平均节点数从 7-14 提升到 20-40
