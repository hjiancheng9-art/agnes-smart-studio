下面按 ComfyFlow Compiler v0.3 / 14 模块 / 3254 行 给清债方案。优先目标：先把编译器核心稳定住，不追求全量 E2E。

1. 当前技术债清单：先修顺序
P0：必须先清
1. Workflow JSON 格式债
   - API Format / Save Format 混用风险
   - node_id 是 str 还是 int 不统一
   - link 输入 ["node_id", output_index] 没有统一校验

2. ComfyUI 环境依赖债
   - /object_info、/models、custom_nodes 扫描没有统一抽象
   - 真实 ComfyUI 不应该出现在单元测试里

3. 蓝图质量债
   - blueprint required_nodes / optional_nodes / required_models 没有合同化
   - Flux / LTX / Wan / SDXL 的 loader mode 没有强约束

4. 参数策略债
   - quality_mode、hardware_budget、task_type 三者可能互相覆盖
   - steps / cfg / resolution / batch_size 需要统一 clamp

5. 错误类型债
   - 编译失败、环境缺失、模型缺失、质量门失败、显存预算失败不能都返回 str
P1：第二批清
6. Golden workflow 缺失
   - 没有固定样本防止 compiler 输出结构漂移

7. 模块边界不清
   - intent / recipe / blueprint / graph compose / validate / quality gate 职责可能混在一起

8. 可复现 manifest 不完整
   - blueprint_id、compiler_version、model_names、node_requirements、quality_report 应固定

9. fallback 行为不可测
   - 缺节点 / 缺模型 / 显存不足时应该有确定降级链

10. 缺少 smoke test
   - 至少要保证 3 个核心任务能编译成功：sd_txt2img、flux_txt2img、ltx_i2v

5. 覆盖率目标

对 3000+ 行编译器项目，建议分阶段：

v0.3 清债阶段：
  total coverage >= 65%
  核心模块 >= 80%
  API / CLI / 适配层可以低一些

v0.4 稳定阶段：
  total coverage >= 75%
  核心模块 >= 85%

v1.0 前：
  total coverage >= 80%
  核心模块 >= 90%

不要一开始把总覆盖率卡到 85%，会逼你写一堆低价值测试。先把 compiler / validator / blueprint / parameter_policy / fallback 测牢。

def test_parameter_policy_clamps_bad_values():
    from comfyflow.parameter_policy import ParameterPolicy
    from comfyflow.task_spec import TaskSpec
    from comfyflow.runtime_budget import RuntimeBudget

    task = TaskSpec(
        intent="txt2img",
        prompt="cat",
        width=9999,
        height=1001,
        steps=100,
        cfg=99,
        batch_size=8,
        aspect_ratio="9:16",
    )

    budget = RuntimeBudget(
        tier="low_6gb",
        vram_gb=6,
        max_width=768,
        max_height=1152,
        max_batch_size=1,
        allow_sdxl=False,
        allow_flux=False,
        allow_video=False,
    )

    spec = ParameterPolicy().apply(task, budget)

    assert spec.width % 8 == 0
    assert spec.height % 8 == 0
    assert spec.width <= 768
    assert spec.height <= 1152
    assert 8 <= spec.steps <= 40
    assert 1 <= spec.cfg <= 15
    assert spec.batch_size == 1

8. Golden workflow 测试骨架

这个很重要。它防止你改 compiler 后 workflow 结构悄悄变坏。

Python
运行
import json
from pathlib import Path


def normalize_workflow(workflow: dict) -> dict:
    """
    去掉 seed / 随机 id / UI 坐标等不稳定字段。
    """
    result = json.loads(json.dumps(workflow))

    for node in result.values():
        inputs = node.get("inputs", {})
        if "seed" in inputs:
            inputs["seed"] = 123456

    return result


def test_golden_sdxl_txt2img(tmp_path):
    from comfyflow.compiler import ComfyFlowCompiler

    compiler = ComfyFlowCompiler.from_fake_environment(
        vram_gb=12,
        models={
            "checkpoints": ["sd_xl_base_1.0.safetensors"],
        },
    )

    result = compiler.compile(
        "电影感赛博朋克猫，霓虹雨夜，9:16",
        quality_mode="balanced",
    )

    actual = normalize_workflow(result.workflow)

    golden_path = Path("tests/fixtures/golden/sdxl_txt2img_basic.json")
    expected = json.loads(golden_path.read_text(encoding="utf-8"))

    assert actual == expected
