A. DeliberateWorkflow → CapabilityRuntimeRouter 集成

DeliberateWorkflow.execute() 不是用 self._plan / self._attack / self._criticize / self._repair 做死循环，而是：

先调 IntelligencePolicyRouter.route() 拿到 mode
再调 CapabilityRuntimeRouter.select_runtime() 拿到 Runtime
然后调 Runtime.execute()
如果有 critic/repair 需求再调 CriticAgent

改成这样：

```
async def execute(self, request, context=None, mode=None):
    # 1. Policy
    mode = mode or self.policy_router.route(request, context)
    config = self.policy_router.get_mode_config(mode)
    
    # 2. Try Capability Runtime first
    ctx = RuntimeContext(request=request, mode=mode.value, config=config.to_dict())
    rt_type, runtime = self.capability_router.select_runtime(request, mode.value)
    
    if runtime and rt_type != CapabilityRuntimeType.GENERAL:
        # 专业 Runtime 执行
        result = await runtime.execute(ctx)
        # 只有需要 critic/repair 时才继续走传统流程
        if config.critic and result.get("status") == "failed":
            pass  # 走 CriticAgent 修复
    else:
        # Fallback: GeneralRuntime（保留原逻辑）
        result = await self.general_runtime.execute(ctx)
```

B. RuntimeConfig 灰度发布控制

```python
@dataclass
class RuntimeConfig:
    enabled: bool = True
    rollout: dict[str, bool] = field(default_factory=lambda: {
        "general": True,
        "debug_analyze": True,
        "architecture": True,
        "creative": True,
        "research": True,
        "code_patch": False,
        "security": False,
    })
    
    def is_runtime_enabled(self, name: str) -> bool:
        return self.enabled and self.rollout.get(name, False)
```

先打开低风险 Runtime, CodePatch 和 Security 最后开。
