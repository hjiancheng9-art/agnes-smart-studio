 检查是否存在测试 import 即启动服务：python -X importtime -m pytest --collect-only tests/test_zcode_launcher.py

 输出第一份测试基线报告：收集数量、通过数量、失败数量、跳过数量、耗时 Top 50。

建议新增 pytest.ini：

INI
[pytest]
testpaths = tests
addopts = -ra --strict-markers
markers =
    unit: fast isolated unit tests
    integration: integration tests with local components
    e2e: end-to-end tests
    slow: tests slower than 2 seconds
    browser: tests requiring browser or CDP
    network: tests requiring network access
    mcp: tests requiring MCP server
    lsp: tests requiring LSP server

python -m pytest --collect-only -q 不再失败。

python -m pytest -m "unit and not slow" --maxfail=1 可在 60 秒内完成。

阶段2: 切清核心边界，定义统一契约

目标：不急着删模块，先给 80+ 工具、Agent、执行器、浏览器、MCP 建统一接口。否则后续补测试会被动态行为拖垮。

 新建核心契约目录：core/contracts/

 定义 ToolSpec：工具名、版本、输入 schema、输出 schema、权限、超时、幂等性、依赖能力。

 定义 ToolResult：ok、data、error、logs、artifacts、metrics。

 定义 ToolError：VALIDATION_ERROR、PERMISSION_DENIED、TIMEOUT、DEPENDENCY_MISSING、RUNTIME_ERROR、REMOTE_ERROR。

 定义 AgentTask：目标、上下文、约束、允许工具、禁止工具、预算、审批策略。

 定义 ExecutionContext：workspace、env、权限、trace_id、deadline、cancel_token。

 定义 BrowserSession 抽象接口，隔离 CDP / Playwright 实现。

 定义 MCPClient 抽象接口，隔离真实 MCP server 和 fake MCP。

 定义 LspSession 抽象接口，隔离真实 LSP 和 fake LSP。

 把 tool_registry_mesh 的输出统一成 ToolSpec。

 把所有工具执行结果统一成 ToolResult，禁止直接返回裸 dict 或裸异常。

 给 approval_gate 增加统一入口：任何高危工具必须通过审批。

 给 sandbox 增加不可绕过的执行入口：任何 shell/python/git 文件写操作必须经过执行上下文。

 给 context_manager 增加 token 预算和消息来源标记，防止历史消息被错误注入输入区。

 给 cdp_browser 增加状态机：DISCONNECTED、CONNECTING、READY、DEGRADED、RECOVERING、FAILED。

 给 task_governor 增加全局任务预算：最大并发、最大重试、最大运行时间、最大工具调用数。
