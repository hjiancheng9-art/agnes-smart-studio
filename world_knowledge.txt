0. 核心公式与总规则
LoRA 的核心思想是冻结原模型权重，只训练低秩增量：
W' = W + ΔWΔW = scale × B × Ascale = alpha / rank
其中 rank/network_dim 控制 LoRA 表达容量，alpha/network_alpha 控制注入强度。原始 LoRA 论文提出冻结预训练权重并注入低秩矩阵，以显著减少可训练参数；实践中 alpha/rank 是你判断 LoRA 更新强度的核心比例。arXiv+1
训练步数公式：
steps_per_epoch = ceil(image_count × repeats / batch_size / gradient_accumulation_steps)total_steps = steps_per_epoch × epochs
推荐先用目标步数倒推 repeats：
repeats = ceil(target_steps × batch_size / (image_count × epochs))
经验目标步数：
角色 LoRA：800 ~ 2500 steps风格 LoRA：1500 ~ 5000 steps物体 LoRA：1000 ~ 3000 steps服装 LoRA：800 ~ 2500 stepsFLUX LoRA：800 ~ 2500 steps 起步，少图不要硬拉太长

1.4 常见训练失败与修复
现象可能原因修复一用 LoRA 脸就崩lr 太高 / rank 太高 / 数据差lr 降 30%，rank 降到 16/32，清理低质图非常像训练图，但不能换姿势过拟合 / caption 太短 / 数据角度单一降 repeats/epochs，加 caption 细节，补不同角度完全不像欠拟合 / trigger 不稳定 / 步数太少增加 steps 30~50%，统一 trigger画面脏、噪点多训练图质量差 / lr 高 / alpha 高清理数据，lr 降到 5e-5，alpha=rank/2风格污染整个画面权重过高 / caption 把背景也学进去了推理权重 0.5~0.8，caption 去掉不想绑定的背景出 NaN / loss 爆炸fp16 VAE / lr 高 / 优化器不稳SDXL 加 --no_half_vae，改 bf16，降 lr训练很慢没缓存 / 分辨率过高开 cache_latents、cache_text_encoder_outputs、降分辨率训练不动 loss 很平lr 太低 / 数据过少lr 提高到 1e-4，增加 repeats

3. 图像处理资料
3.1 训练图片预处理流程
导入图片↓格式转换 PNG/JPEG↓去重↓质量检测↓主体检测 / 裁剪↓分辨率归一↓caption↓train/validation/test 划分
推荐规则：
格式：PNG 或高质量 JPG最短边：SD1.5 ≥ 512，SDXL ≥ 768，最好 1024+人脸角色：脸部不要小于图像高度 20%风格 LoRA：构图和题材要多样角色 LoRA：角度、表情、服装、背景要多样
3.2 去重方法与阈值
pHash
距离公式：
d = HammingHash(phash(image_a), phash(image_b))
阈值：
d <= 4：几乎重复5 <= d <= 8：高度相似9 <= d <= 12：可能相似，需要人工看d > 12：通常可保留
SSIM
SSIM 比 MSE 更接近人眼感知，因为它比较结构信息；scikit-image 提供 structural_similarity 函数，并提醒 float 图像要注意 data_range。Scikit-Image+1
公式简写：
SSIM(x,y) = ((2μxμy + C1)(2σxy + C2)) / ((μx² + μy² + C1)(σx² + σy² + C2))
阈值：
SSIM >= 0.95：近重复0.90~0.95：高度相似0.80~0.90：相似构图<0.80：通常不同
CLIP similarity
公式：
sim = cosine(clip_image_embedding_a, clip_image_embedding_b)
阈值：
>= 0.98：几乎重复0.95~0.98：高度相似0.90~0.95：同主体/同构图<0.90：通常保留
智能体建议：
先 pHash 快速去重再 SSIM 判断近似构图最后 CLIP 判断语义重复

参数含义推荐范围seed随机种子固定复现；-1 随机steps采样步数20~35 常规；极速 4~12；高质量 35~50cfgprompt 引导强度SD1.5: 6~9；SDXL: 4.5~7；FLUX: 通常更低或按节点sampler_name采样器euler, dpmpp_2m, dpmpp_2m_sde, dpmpp_3m_sdescheduler调度器normal, karras, exponential, sgm_uniformdenoise重绘强度txt2img=1.0；img2img=0.25~0.75；高清修复=0.15~0.35
推荐组合：
SD1.5 通用：DPM++ 2M Karras, steps 24~32, cfg 7SDXL 通用：DPM++ 2M SDE Karras, steps 25~35, cfg 5~7动漫：Euler a / DPM++ 2M Karras, cfg 6~8写实：DPM++ 2M SDE Karras, cfg 4.5~6.5LCM/Turbo：steps 4~8, cfg 1~2.5FLUX：按 FluxGuidance / 专用节点，steps 20~30 起步

4.2 常见画质问题与参数修复
问题调整太模糊steps +5~10；换 DPM++ 2M SDE；提高分辨率；降低 denoise太暗prompt 加 lighting；cfg +0.5；后处理曝光；换 VAE变形cfg -1；steps +5；ControlNet/IPAdapter 约束；换 seed颜色不对降低 LoRA 权重；降低 cfg；加明确色彩 prompt噪点多steps +5；降 cfg；换 sampler；检查 VAE过饱和cfg -1；LoRA strength -0.1；prompt 去掉 extreme/vivid细节脏降 LoRA 权重；使用高清修复 denoise 0.2~0.3不像参考图IPAdapter weight +0.1；LoRA strength +0.1；固定 seed 对比动作太小视频 motion strength +0.1~0.2；prompt 写清运动视频闪烁降 motion；减少 prompt 冲突；缩短 duration；固定 seed

4.3 LTXV 视频参数
LTX-Video 官方模型卡称其为 DiT-based video generation model，可生成 30FPS、1216×704 高质量视频；ComfyUI 示例页面也强调 LTX-Video 需要长而具体的描述性 prompt。Hugging Face+1
起始参数：
resolution: 768x512 / 832x480 / 1216x704fps: 24 或 30duration: 3~5 秒起步frames: fps × durationsteps: 20~40cfg/guidance: 3~7，视节点而定motion strength: 0.4~0.8seed: 固定用于复现prompt: 长描述，包含主体、动作、镜头、场景、光线
视频 prompt 模板：
[主体] 在 [场景] 中 [动作]，镜头 [camera motion]，光线 [lighting]，画面风格 [style]，运动连续、主体稳定、无闪烁、无形变。

6. 测试验证资料
6.1 LoRA 权重对比矩阵
训练完成后必须自动跑：
权重：0.3 / 0.5 / 0.7 / 0.9 / 1.1seed：固定 3 个prompt：固定 6 条总图数：5 × 3 × 6 = 90 张
角色 LoRA 测试 prompt：
1. sks_char, portrait photo, natural light, looking at camera2. sks_char, full body, standing, city street, cinematic lighting3. sks_char, side view, outdoor, soft light4. sks_char, different outfit, white shirt, studio photo5. sks_char, smiling, close-up portrait6. sks_char, action pose, dramatic lighting
风格 LoRA 测试 prompt：
1. sks_style, a house in the forest2. sks_style, portrait of a woman3. sks_style, futuristic city at night4. sks_style, product photo of a perfume bottle5. sks_style, wide landscape, sunset6. sks_style, simple object on white background
评分维度：
identity_score: 像不像目标style_score: 风格迁移度prompt_adherence: 提示词遵循artifact_score: 崩坏/噪点/脏图overfit_score: 是否只会训练图姿势/服装best_weight: 推荐推理权重

6.2 X/Y/Z Plot 方法
推荐维度：
X: LoRA weight = 0.4 / 0.6 / 0.8 / 1.0Y: checkpoint epoch = 2 / 4 / 6 / 8 / 10Z: seed = 1 / 2 / 3
判断：
低 epoch 不像，高 epoch 崩：中间 checkpoint 最好低 weight 不像，高 weight 污染：最佳权重在 0.6~0.8所有 seed 都不像：数据/caption/训练不足只有某些 seed 好：LoRA 不稳定，需要更多数据或降 lr

8. 最终优先级
你要把这份知识库接进智能体，建议顺序是：
1. ComfyUI API 执行器知识：/prompt /history /upload /view /ws2. workflow_patterns：txt2img/img2img/inpaint/controlnet/lora/upscale3. sampler_profiles：KSampler 参数调优4. LoRA training_profiles：SD1.5/SDXL/FLUX 默认训练配置5. preprocess_rules：去重、裁剪、caption6. evaluation_profiles：X/Y/Z plot 和 checkpoint 评估7. error_fixes：训练失败和工作流失败自动修复
这样你的智能体就不是“只会套模板”，而是能做到：
用户目标↓选择训练/推理模式↓准备数据↓生成配置↓执行训练或工作流↓监控状态↓测试验证↓自动调参↓集成回 ComfyUI 工作流
