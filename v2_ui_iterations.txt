最低 2 轮稳妥 3 轮

成熟度可到：
82 / 100

目标：从“后台控制台”升级成“品牌驾驶舱”。

这一轮完成后，可以给真实用户试用。

3. 只做 3 个体验改进，优先做哪三个？
只做三件，我会选这三个。

第一件：新手引导 + 当前最佳下一步
这是最重要的。
现在功能很多，用户进来不知道先点哪。
驾驶舱首页必须直接告诉用户：
当前项目状态是什么下一步应该做什么为什么不能继续点哪个按钮可以继续
建议做一个 Next Action 面板：
当前推荐操作：运行 3-keyframes → ComfyUI 出关键帧原因：剧本和分镜已完成，缺少真实关键帧制品。需要：- ComfyUI 在线- styleAnchor 已锁定- storyboard shots 已存在按钮：[一键运行关键帧生成][查看缺失项][切换执行平台]
这一个功能能大幅降低使用门槛。
优先级：
P0
加分最高：
+3 到 +4 分

第二件：错误提示人话化 + 修复按钮
现在错误还是 JSON 格式，这是产品感最大杀手。
不要直接显示：
JSON{  "code": "quality_gate_blocked",  "stageId": "3-keyframes",  "details": {}}
应该显示：
关键帧阶段被质量门阻断原因：当前镜头 shot_003 没有真实图片文件，不能进入动画阶段。建议修复：1. 重新运行 ComfyUI 关键帧生成2. 或手动上传 shot_003 图片3. 或跳过该镜头并重新生成分镜操作：[重新运行][手动上传][查看详情]
错误提示要分三层：
用户可读摘要技术详情折叠一键修复动作
优先级：
P0
加分：
+2 到 +3 分

第三件：常用工作流 / 收藏 / 最近使用
你现在有工作流搜索，但没有历史和收藏。
这会导致用户每次都重新找。
建议做：
最近运行常用工作流收藏模板推荐模板失败后建议模板
特别是 V2 + ComfyUI 场景，用户最需要的是：
上次那个能出图的模板这个项目常用的模板这个阶段推荐的模板
首页可以放：
常用入口：[文字生成关键帧][分镜生成图][角色设计][LoRA训练][ComfyUI模板推荐][完整24阶段一键流]
优先级：
P1
加分：
+2 分

为什么不把“实时性”放前三？
8秒轮询确实不优雅，但你已经有执行 + SSE。
如果当前关键执行流已经能实时显示，WebSocket 不是最急。
我会这样判断：
SSE 已够用：P18秒轮询全局面板：可接受真正要紧的是用户看不懂、错误看不懂、下一步不知道点哪
WebSocket 可以后做，除非你有多人协作、任务并发、实时日志流强需求。

为什么不把“用户系统”放前三？
用户系统重要，但它不是当前单机/本地 V2 的最短板。
当前要先解决：
第一次使用能不能跑起来出错后能不能修常用流程能不能复用
用户系统属于产品化第二阶段或第三阶段。
对本地 CRUX 总控型工具来说，它不是眼前最阻塞的体验问题。
