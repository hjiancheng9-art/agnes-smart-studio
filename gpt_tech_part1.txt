"""
ComfyUI 有两种常见 JSON：

1) UI Save Format
   - 用于前端完整保存
   - 包含 nodes、links、groups、pos、size、color、widgets_values 等 UI 信息
   - 适合重新打开编辑

2) API Format / Prompt Format
   - 用于 /prompt API 执行
   - 根对象是 dict
   - key 是字符串形式 node_id，例如 "3"、"4"、"5"
   - 每个节点必须包含：
       {
         "class_type": "...",
         "inputs": {...}
       }
   - 可选字段：
       "_meta": {"title": "..."}
   - 连线格式：
       某个 input 如果来自另一个节点输出，写成 ["node_id", output_index]
       例如：
       "model": ["4", 0]
       表示读取 4 号节点的第 0 个输出

提交到 ComfyUI API 时一般这样：
    POST http://127.0.0.1:8188/prompt
    body:
    {
      "prompt": SDXL_TXT2IMG_WORKFLOW_API
    }

_meta 作用：
    - 非执行必需
    - 主要给 UI / 调试 / 可读性使用
    - 常见内容是 {"title": "KSampler"}
    - 删除 _meta 通常不影响执行
"""
