### 三方核心共识

1. **分层七兽（B方案）为首选** -- 装饰用七兽品牌色，内容用常规语义色
2. **颜色必须绑定文字前缀标签** -- [U]/[A]/[S]/[E] 等标记
3. **需要三主题** -- normal / high_contrast / mono
4. **动画约束** -- 同时最多1个动效，流式输出时全关
5. **七兽是品牌资产，不是纯功能性色彩系统**

## 综合总结

### 三方一致的核心共识

- **七兽配色**：分层策略：装饰用七兽品牌色，内容用常规语义色
- **布局**：Dashboard 默认隐藏/浮动，大屏展开；消息区优先级最高
- **输入体验**：默认单行自动扩展；Enter 发送为主，Alt+Enter 换行
- **性能**：必须做虚拟滚动 + 增量渲染缓存；动画帧率统一治理
- **可访问性**：颜色必须绑定文字符号前缀；提供降级模式
- **消息渲染**：只支持 AI 编程最常用的 20% Markdown
- **动画**：同时最多 1 个动效；流式输出时全关
- **远程/SSH**：自动检测 SSH 环境并降级；最小支持 80x24
- **整体定位**：TUI 是开发者驾驶舱，GUI 是创意生产驾驶舱

### 最终 Top 5 改进建议

1. **P0 -- 响应式布局**：Dashboard 按终端宽度自动切换，默认隐藏/浮动，大屏展开
2. **P0 -- 降级策略**：动画/颜色/复杂度按环境自动降级（SSH/小窗口/色盲）
3. **P1 -- 虚拟滚动 + 增量渲染**：只渲染可见区域，缓存已渲染消息
4. **P1 -- 问题导向 Dashboard**：平时安静，出事说话
5. **P2 -- TUI+GUI 双形态**：共享后端协议，按场景切换前端形态
