# Test-Driven Development
## Description
Write tests FIRST, then code. Red-Green-Refactor cycle.
## Instructions
1. RED: Write a failing test that defines the expected behavior
2. GREEN: Write minimal code to make the test pass
3. REFACTOR: Clean up code while keeping tests green
4. Never write code without a failing test first
5. Run python -m pytest tests/ -q after every change
6. Keep tests fast (>50 in <2s), isolated, and deterministic
