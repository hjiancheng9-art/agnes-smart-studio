# Code Simplification
## Description
Refactor for clarity. Remove complexity without changing behavior.
## Instructions
1. Every line of code should have a clear purpose — delete dead code
2. Extract complex conditions into named functions
3. Replace magic numbers with named constants
4. DRY: if you write it twice, extract it — but don't over-abstract
5. Prefer readability over cleverness
6. Single Responsibility: each function does one thing well
7. Short functions (<30 lines ideal)
8. Run tests after every simplification step
