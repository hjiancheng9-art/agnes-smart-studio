# Source-Driven Development
## Description
Ground every implementation decision in official documentation.
## Instructions
1. Before writing code, read the official docs for the library/framework
2. Cite specific documentation sections in code comments
3. Prefer documented APIs over undocumented workarounds
4. If docs are unclear, read the source code of the library
5. Write code that matches the documented patterns and conventions
6. Never guess API behavior -- verify with docs or REPL
