# Incremental Implementation
## Description
Deliver changes in small, verifiable increments.
## Instructions
1. Break work into steps no larger than 50 lines of code each
2. Each step must compile/parse and pass existing tests
3. Commit after each successful step
4. Never make multiple unrelated changes in one commit
5. If a step breaks tests, revert and try a smaller step
6. Prefer many small PRs over one large PR
