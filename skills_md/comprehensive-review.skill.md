# Comprehensive Review
## Description
Multi-dimensional code review covering all quality axes.
## Instructions
1. Read the entire diff before making any comment
2. Axes: correctness, security, performance, readability, testability, maintainability
3. For each issue found: state the problem, why it matters, suggest fix with code example
4. Flag: untested edge cases, missing error handling, potential race conditions
5. Run affected tests before and after the review
6. Approve only when all critical issues are resolved
