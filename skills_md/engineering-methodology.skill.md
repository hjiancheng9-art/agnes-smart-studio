# Engineering Methodology
## Description
## 工程方法论（内化自 Codex 七轮实战）

以下模式不是教条，是经过验证的有效工作方式。面临任何问题时，按此框架行动。

### 诊断原则
- **先收集证据，再下结论** — 读日志、读错误文件、读 traces、读最近改动。不猜。
- **区分根因和症状** — 503 不是"修 HTTP 调用"，是"发现模型被下架了"。provider_switcher 模块不存在不是"换条路"，是"创建缺失模块"。
- **全局扫描，非点状修复** — 发现一个 silent except-pass，search_files 搜全库找同类。改一处，防一类。

### 行动原则
- **最小改动，最大效果** — 不要重构无关代码，精准打击根因。
- **工具优先，人工其次** — 能用 ast.parse 验证就用，能用 pytest 跑就用，不用肉眼检查。
- **改前快照，改后验证** — 每次修改后立即验证语法和测试，不信任自己的修改。

### 优先级判断
1. 会导致崩溃/不可用的 → 立刻修
2. 会静默错误/误导决策的 → 高优先级
3. 功能缺失但可降级运行的 → 中优先级
4. 纯代码整洁 → 顺带修，不单独做

### 自检节奏
每次 `/self audit` 应该产生 0 个严重/高危发现。如果出现了，先修完再干别的。
每次修改文件后，跑 `python -c "import ast; ast.parse(...)"` 验证语法。
每次会话结束前，跑 `python -m pytest tests/ -q` 确认零回归。
