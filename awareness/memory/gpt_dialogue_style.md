# ChatGPT 对话风格铁律

## 核心要求：正反方思辨逻辑

和 ChatGPT 聊架构/代码问题时，必须采用 **正反方思辨** 模式：

### 原则
1. **不要单纯求答案** — 不要发一个软问题让 GPT 给通用建议
2. **正反方对撞** — 每个方案自己先拆成正方/反方立场，再让 GPT 裁决
3. **预设反对立场** — 对 GPT 给的任何方案，主动找反方论据反驳它
4. **不要让 GPT 抹平逻辑** — 它倾向于给出圆滑的 consensus 方案，要逼它亮出真实 trade-off

### 提问模板
```
正反方：
Bug X — [一句话描述]
正方：[为什么这么做]
反方：[为什么不这么做]
倾向：[你倾向哪边]
裁决：请裁定
```

### 交锋方式
- "反方可能会说…，你怎么回应？"
- "你说的方案在场景 A 可行，但场景 B 下会怎样？"
- "正方这个论点有漏洞：…"
- "你倾向的方案有没有你不知道的坑？"
- "我不接受折中方案，选一边，为什么？"

### 禁止
- ❌ "帮我看看"（太软）
- ❌ "有什么建议"（太泛）
- ❌ 不带立场地抛问题
- ❌ 接受模糊的"都可以"答案

### 目的
不是让 GPT 帮你"解决问题"，而是让 GPT 和你一起 **逼出问题的真实边界**。每次聊完要有明确的"知道了什么"和"还不知道什么"。
